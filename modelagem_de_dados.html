Documentação de Modelagem de Dados

1. Modelo Entidade-Relacionamento (MER)

O MER (Modelo Entidade-Relacionamento) representa a estrutura conceitual dos dados, identificando as entidades, seus atributos e os relacionamentos entre elas.

Entidades Principais

USUÁRIO: Entidade central do sistema (RF001).

DISCIPLINA: Entidade gerenciada pelo usuário (RF004).

ATIVIDADE: Entidade principal a ser registrada, associada a uma disciplina (RF005, RF006).

TIPO_ATIVIDADE e STATUS_ATIVIDADE: Entidades auxiliares para categorização (RF006, RF007).

Relacionamentos

USUÁRIO (1) : N (DISCIPLINA): Um usuário possui múltiplas disciplinas. Uma disciplina pertence a um único usuário.

DISCIPLINA (1) : N (ATIVIDADE): Uma disciplina pode ter múltiplas atividades. Uma atividade pertence a uma única disciplina.

TIPO_ATIVIDADE (1) : N (ATIVIDADE): Um tipo (e.g., Prova) pode ser associado a múltiplas atividades. Uma atividade tem um único tipo.

STATUS_ATIVIDADE (1) : N (ATIVIDADE): Um status (e.g., A Fazer) pode ser associado a múltiplas atividades. Uma atividade tem um único status.
