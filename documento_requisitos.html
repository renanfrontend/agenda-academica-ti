Documentação de Requisitos do Sistema

1. Requisitos Funcionais (RF)

Os Requisitos Funcionais (RF) descrevem o que o sistema deve fazer (as funções e comportamentos).

  • RF001: O usuário deve poder se cadastrar no sistema fornecendo nome, e-mail e senha.
  • RF002: O usuário deve poder fazer login no sistema com e-mail e senha.
  • RF003: O sistema deve exibir um painel (dashboard) com uma visualização de calendário (mensal/semanal).
  • RF004: O usuário deve poder cadastrar, editar e excluir Disciplinas (ex: "Engenharia de Software", "Banco de Dados").
  • RF005: O usuário deve poder cadastrar, editar e excluir Atividades (ex: "Prova", "Trabalho", "Projeto", "Laboratório").
  • RF006: Ao cadastrar uma atividade, o usuário deve poder associá-la a uma disciplina, definir uma data de entrega, um tipo e um status.
  • RF007: A atividade deve ter um Status (ex: "A Fazer", "Em Andamento", "Concluído", "Enviado"). (Diferencial)
  • RF008: O sistema deve exibir um "kanban" simples na dashboard para o usuário arrastar atividades entre os status. (Diferencial)
  • RF009: O sistema deve exibir uma lista de atividades futuras (próximos 7 dias) na dashboard.
  • RF010: O sistema deve enviar notificações (lembretes) ao usuário sobre atividades com prazo próximo (ex: 48h antes).

2. Requisitos Não Funcionais (RNF)

Os Requisitos Não Funcionais (RNF) descrevem como o sistema deve operar, focando em qualidades e restrições (ex: desempenho, segurança, usabilidade).

  • RNF001: A interface deve ser limpa, intuitiva e responsiva, adaptando-se a desktops e dispositivos móveis. (Usabilidade)
  • RNF002: O painel principal (dashboard) e o calendário devem carregar em menos de 2 segundos. (Desempenho)
  • RNF003: O sistema deve ser compatível com as versões mais recentes dos navegadores Google Chrome, Mozilla Firefox e Microsoft Edge. (Compatibilidade)
  • RNF004: As senhas dos usuários devem ser armazenadas de forma criptografada (hashed) no banco de dados. (Segurança)
  • RNF005: A comunicação entre o cliente e o servidor deve ser feita via HTTPS. (Segurança)
  • RNF006: O código-fonte deve seguir padrões de projeto (ex: MVC ou arquitetura de componentes) para facilitar futuras atualizações. (Manutenibilidade)
